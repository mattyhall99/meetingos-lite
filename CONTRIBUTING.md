# Contributing to MeetingOS Lite

Thanks for your interest in improving MeetingOS Lite.

## Reporting Issues

Open an issue with:
- What you expected to happen
- What actually happened
- The type of meeting notes you were processing (Gemini, Otter, Zoom, pasted text, etc.)
- Any error messages or unexpected output

Do not include confidential meeting content in issues. Anonymise or redact before sharing.

## Suggesting Improvements

Ideas for new features, better extraction, or improved formatting are welcome. Open an issue tagged `enhancement` with a clear description of the improvement and why it matters.

## Pull Requests

1. Fork the repo
2. Create a feature branch (`git checkout -b improve-action-extraction`)
3. Make your changes to the SKILL.md file
4. Test against a real meeting transcript (Gemini, Otter, or pasted text)
5. Submit a PR with a description of what changed and why

### What makes a good PR

- Focused on a single improvement
- Tested against at least one real transcript
- Does not break existing functionality
- Follows the existing writing style (active voice, short sentences, no filler)

## Code of Conduct

Be kind. Be constructive. Assume good intent.
